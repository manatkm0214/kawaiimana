# 仕様書

## 1. 目的

家計の入力から分析、予算調整、生活支援までを一元化し、日々の意思決定を支援する。

## 2. 想定利用者

- 個人の家計を管理したいユーザー
- 生活費、固定費、貯蓄、投資のバランスを見たいユーザー
- AI に相談しながら家計改善を進めたいユーザー

## 3. 提供機能

### 3.1 家計入力

- 収入 / 支出 / 貯蓄 / 投資の登録
- カテゴリ、日付、固定費フラグ、メモ入力
- 最近の入力履歴の参照

### 3.2 ダッシュボード

- 当月収支
- カテゴリ別集計
- 予算消化率
- 安全度 / 生活レベル
- 固定費見直し候補

### 3.3 予算管理

- 配分プリセット適用
- 予算超過 / 余剰の確認
- トレードオフ提案
- 引き落とし予約の記録

### 3.4 AI 機能

- AI 分析
- AI 節約アドバイス
- AI チャット相談
- AI 生活支援
- AI 店案内

### 3.5 認証

- Google ログイン
- LINE ログイン
- メール系認証
- ログアウト

### 3.6 その他

- 問い合わせ送信
- 設定変更
- 法務ページ表示
- PDF 出力

## 4. 画面仕様

### 4.1 ホーム

- ログイン前導線
- ニックネーム / 初期導線
- 認証開始

### 4.2 ダッシュボード

- 入力、ボード、カレンダー、目標、AI、年次、基準、シニア、こどもを切替

### 4.3 設定

- デザイン設定
- アカウント設定

### 4.4 問い合わせ

- 名前、メール、件名、本文を送信

## 5. API 仕様

代表 API:

- `/api/transactions`
- `/api/profile`
- `/api/preset`
- `/api/budget-surplus`
- `/api/budget-tradeoff`
- `/api/debit-reservations`
- `/api/ai`
- `/api/ai/chat`
- `/api/nearby-shops`
- `/api/contact`

共通仕様:

- JSON リクエスト
- 認証必須 API は Auth0 セッションを確認
- 更新系 API は same-origin を要求
- 一部 API は rate limit 付き

## 6. AI 店案内仕様

入力:

- 現在地またはエリア文字列
- 店種
- 自由入力キーワード
- 半径

出力:

- 店舗名
- 住所
- 距離
- 緯度経度
- 地図遷移リンク

検索仕様:

- エリア入力時はサーバーで Google Geocoding API（または Nominatim フォールバック）でジオコーディング
- 店舗検索は Google Places API を優先し、失敗時は Overpass API（OSM）へ自動フォールバック
- 近い順に 6 件まで返却
- 検索結果は 5 分間クライアントキャッシュ

エラー仕様:

- エリア不明時は `404`
- ジオコーディング失敗時は `502`
- 不正入力時は `400`

## 7. セキュリティ仕様

- 認証済みユーザーのみ個人データを更新可能
- same-origin 対象 API は cross-site 送信を拒否
- rate limit 対象 API は短時間の多重送信を制限
- 外部向け webhook は署名検証を必須化
