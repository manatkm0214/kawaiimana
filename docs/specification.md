# 仕様書

## 1. 目的

家計の入力から分析、予算調整、目標管理、生活支援までを一元化し、日々の意思決定を助ける。

## 2. 想定利用者

- 個人の家計を継続的に管理したいユーザー
- 固定費、変動費、貯蓄、投資のバランスを確認したいユーザー
- AI に相談しながら予算改善を進めたいユーザー
- 大きな買い物やローン判断を慎重に行いたいユーザー
- こども向け、シニア向けの見やすい表示も使い分けたいユーザー

## 3. 提供機能

### 3.1 家計入力

- 収入 / 支出 / 貯蓄 / 投資の登録
- 金額、日付、カテゴリ、支払い方法、メモ、固定費フラグの入力
- 最近の履歴参照

支出カテゴリ（2026-04-27 時点）:

- 食費 / 住居 / 水道・光熱費 / 通信費 / 交通費 / 医療費 / 日用品 / 娯楽 / レジャー / 趣味
- 教育 / 自己投資 / 保険 / 税金 / 交際費 / サブスク / ペット / 美容・衣服 / ゲーム / 推し活
- 寄付・支援 / その他

### 3.2 ダッシュボード

- 当月収支
- カテゴリ別集計
- 予算消化率
- 安全度 / 生活レベル
- 固定費見直し候補
- AI 支援ウィジェット

### 3.3 目標管理

目標領域は「個人 / こども / シニア」の世代切替を持つ。

一般向け `GoalsAndDebt` の機能:

- ローン / 借入管理
- 先取積み立て管理
- 個人目標管理
- 固定費見直し候補のレビュー
- 徳ポイントとチケット管理

### 3.4 ローン / 借入

- ローン種別: `loan`, `mortgage`, `credit`, `other`
- 残高、借入総額、月返済、年利、メモを保持
- 返済率と完済予定月を表示
- 「返済」操作で残高を月返済額ぶん減算

### 3.5 先取積み立て

- 名前、絵文字、目標金額、現在額、目標月、メモを保持
- 追加、編集、積立加算、達成後の過去化、復元、削除
- 未達成時は残額と月次必要額を表示

### 3.6 個人目標

- 名前、絵文字、目標金額、現在額、メモを保持
- 手動加算とチケット加算に対応
- 達成後の過去化、復元、削除に対応
- 購入アドバイザーから「目標に追加」で連携可能

### 3.7 固定費見直し

- 当月の支出から固定費候補を抽出
- 優先度 `high / medium / low` を付与
- 理由、月額影響、支払い方法、最終発生日、推奨アクションを表示
- 「確認済み」と「対応済みで非表示」に対応
- 非表示は月単位で管理し、「今月非表示にした N 件を再表示」で戻せる

### 3.8 徳ポイント

- 取引履歴を月単位で評価し、徳ポイントを算出
- 1 か月ごとの内訳と合計ポイントを表示
- 合計ポイント `100` ごとにチケット `1` 枚を獲得
- チケット価値はユーザーが変更可能
- 個人目標にチケットを使って進捗を加算可能

### 3.9 購入アドバイザー

- 商品カテゴリと価格を入力して購入判断を支援
- 労働日数と貯蓄月数を算出
- 車 / 住宅は PMT 公式でローン月額を算出
- AI 相場クエリで価格目安を取得
- 個人目標への追加に対応

### 3.10 AI 機能

- AI 分析
- AI 節約アドバイス
- AI チャット相談
- AI 生活支援
- AI 店案内
- AI 相場クエリ

### 3.11 認証

- Google ログイン
- LINE ログイン
- メール系認証
- ログアウト

### 3.12 その他

- 問い合わせ送信
- 設定変更
- 法務ページ表示
- PDF 出力

- `/api/nearby-shops`

共通仕様:

- JSON リクエスト
- 認証必須 API は Auth0 セッションを確認
- 更新系 API は same-origin を要求
- 一部 API は rate limit 付き

## 6. AI 店案内仕様

入力:

- 現在地またはエリア文字列
- 店種
- 自由入力キーワード
- 半径

出力:

- 店舗名
- 住所
- 距離
- 緯度経度
- 地図遷移リンク

検索仕様:

- エリア入力時は Google Geocoding API を優先し、失敗時は Nominatim へフォールバック
- 店舗検索は Google Places API（New）を優先し、失敗または 0 件時は Overpass API へフォールバック
- Overpass 障害時は予備エンドポイントへ切替
- 近い順に最大 6 件まで返却

## 7. 購入アドバイザー仕様

入力:

- カテゴリ: `daily / appliance / beauty / game / oshi / car / home`
- 購入価格（円）

計算出力:

- 労働日数: `価格 ÷ (手取り月収 ÷ 30)`
- 貯蓄月数: `価格 ÷ 月次貯蓄額`
- ローン月額（car / home のみ）
- 購入可能ライン判定

アフォーダビリティ閾値:

- home: ローン月額 ≤ 手取り月収の 25%
- car: ローン月額 ≤ 手取り月収の 20%
- その他: 購入価格 ≤ 月次余剰

目標連携:

- 保存先キー: `kakeibo-personal-goals`
- 保存形式: `{ id, name, emoji, targetAmount, currentAmount, memo, status, createdAt }`

## 9. セキュリティ仕様

- 認証済みユーザーのみ個人データ更新可能
- same-origin 対象 API は cross-site 送信を拒否
- rate limit 対象 API は短時間の多重送信を制限
- 外部向け webhook は署名検証を必須化
