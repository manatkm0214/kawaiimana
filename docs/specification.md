# 仕様書

## 1. 目的

家計の入力から分析、予算調整、生活支援までを一元化し、日々の意思決定を支援する。

## 2. 想定利用者

- 個人の家計を管理したいユーザー
- 生活費、固定費、貯蓄、投資のバランスを見たいユーザー
- AI に相談しながら家計改善を進めたいユーザー
- 購入判断に迷っているユーザー（大きな買い物・ローン判断も含む）

## 3. 提供機能

### 3.1 家計入力

- 収入 / 支出 / 貯蓄 / 投資の登録
- カテゴリ、日付、固定費フラグ、メモ入力
- 最近の入力履歴の参照

支出カテゴリ（2026-04-25 時点）:

- 食費 / 住居 / 水道・光熱費 / 通信費 / 交通費 / 医療費 / 日用品 / 娯楽 / レジャー / 趣味
- 教育 / 自己投資 / 保険 / 税金 / 交際費 / サブスク / ペット / 美容・衣服 / ゲーム / 推し活
- 寄付・支援 / その他

### 3.2 ダッシュボード

- 当月収支
- カテゴリ別集計
- 予算消化率
- 安全度 / 生活レベル
- 固定費見直し候補

### 3.3 予算管理

- 配分プリセット適用
- 予算超過 / 余剰の確認
- トレードオフ提案
- 引き落とし予約の記録

### 3.4 購入アドバイザー

- 商品カテゴリと購入価格を入力して購入判断を支援
- 労働日数・貯蓄月数で「どれくらいの努力が必要か」を数値化
- 家・車はローン月額（PMT 公式）と収入比で購入可能ラインを判定
- その他は月次余剰で購入可能ラインを判定
- AI 相場クエリで市場価格の目安を確認
- 「目標に追加」で個人目標タブと連携して進捗管理

### 3.5 AI 機能

- AI 分析
- AI 節約アドバイス
- AI チャット相談
- AI 生活支援
- AI 店案内
- AI 相場クエリ（購入アドバイザー内）

### 3.6 認証

- Google ログイン
- LINE ログイン
- メール系認証
- ログアウト

### 3.7 その他

- 問い合わせ送信
- 設定変更
- 法務ページ表示
- PDF 出力

## 5. API 仕様

- `/api/budget-tradeoff`

- `/api/ai/chat`
- `/api/nearby-shops`

共通仕様:

- JSON リクエスト
- 認証必須 API は Auth0 セッションを確認
- 更新系 API は same-origin を要求
- 一部 API は rate limit 付き

`POST` API ルート数: `19`（same-origin 適用: `16`、rateLimit 適用: `7`）

## 6. AI 店案内仕様

入力:

- 現在地またはエリア文字列
- 店種
- 自由入力キーワード
- 半径

出力:

- 店舗名
- 住所
- 距離
- 緯度経度
- 地図遷移リンク

検索仕様:

- エリア入力時はサーバーで Google Geocoding API（または Nominatim フォールバック）でジオコーディング
- 店舗検索は Google Places API（New）を優先し、失敗または 0 件の場合は Overpass API（OSM）へ自動フォールバック
- Overpass 障害時は予備エンドポイント（overpass.kumi.systems）へ自動切り替え
- 近い順に 6 件まで返却
- 0 件時はエラーコードと検索座標を診断情報としてレスポンスに含む

エラー仕様:

- エリア不明時は `404`
- ジオコーディング失敗時は `502`
- 不正入力時は `400`

## 7. 購入アドバイザー仕様

入力:

- カテゴリ（daily / appliance / beauty / game / oshi / car / home）
- 購入価格（円）

計算出力:

- 労働日数: `価格 ÷ (手取り月収 ÷ 30)`
- 貯蓄月数: `価格 ÷ 月次貯蓄額`
- ローン月額（car / home のみ、PMT 公式）
- 購入可能ライン判定（アフォーダビリティ）

アフォーダビリティ閾値:

- home: ローン月額 ≤ 手取り月収の 25%
- car:  ローン月額 ≤ 手取り月収の 20%
- その他: 購入価格 ≤ 月次余剰（収入 − 支出）

目標連携:

- 「目標に追加」ボタンで `kakeibo-personal-goals`（localStorage）へ保存
- 保存形式: `{ id, name, emoji, targetAmount, currentAmount: 0, memo, status: "active", createdAt }`

## 8. セキュリティ仕様

- 認証済みユーザーのみ個人データを更新可能
- same-origin 対象 API は cross-site 送信を拒否
- rate limit 対象 API は短時間の多重送信を制限
- 外部向け webhook は署名検証を必須化

セキュリティ評価: `86%`（2026-04-25 実施）
