仕様書

仕様書

1. 目的

家計の入力から分析、予算調整、生活支援までを一元化し、日々の意思決定を支援する。

2. 想定利用者

  • 個人の家計を管理したいユーザー
  • 生活費、固定費、貯蓄、投資のバランスを見たいユーザー
  • AI に相談しながら家計改善を進めたいユーザー
  • 購入判断に迷っているユーザー（大きな買い物・ローン判断も含む）

3. 提供機能

3.1 家計入力

  • 収入 / 支出 / 貯蓄 / 投資の登録
  • カテゴリ、日付、固定費フラグ、メモ入力
  • 最近の入力履歴の参照

支出カテゴリ（2026-04-25 時点）:

  • 食費 / 住居 / 水道・光熱費 / 通信費 / 交通費 / 医療費 / 日用品 / 娯楽 / レジャー / 趣味
  • 教育 / 自己投資 / 保険 / 税金 / 交際費 / サブスク / ペット / 美容・衣服 / ゲーム / 推し活
  • 寄付・支援 / その他

3.2 ダッシュボード

  • 当月収支
  • カテゴリ別集計
  • 予算消化率
  • 安全度 / 生活レベル
  • 固定費見直し候補

3.3 予算管理

  • 配分プリセット適用
  • 予算超過 / 余剰の確認
  • トレードオフ提案
  • 引き落とし予約の記録

3.4 購入アドバイザー

  • 商品カテゴリと購入価格を入力して購入判断を支援
  • 労働日数・貯蓄月数で「どれくらいの努力が必要か」を数値化
  • 家・車はローン月額（PMT 公式）と収入比で購入可能ラインを判定
  • その他は月次余剰で購入可能ラインを判定
  • AI 相場クエリで市場価格の目安を確認
  • 「目標に追加」で目標画面の「個人」タブ内の個人目標と連携して進捗管理

3.5 AI 機能

  • AI 分析
  • AI 節約アドバイス
  • AI チャット相談
  • AI 生活支援
  • AI 店案内
  • AI 相場クエリ（購入アドバイザー内）

3.6 認証

  • Google ログイン
  • LINE ログイン
  • メール系認証
  • ログアウト

3.7 その他

  • 問い合わせ送信
  • 設定変更
  • 法務ページ表示
  • PDF 出力

4. 画面仕様

4.1 ホーム

  • ログイン前導線
  • ニックネーム / 初期導線
  • 認証開始

4.2 ダッシュボード

  • 入力、ボード、カレンダー、目標、AI、年次、基準、シニア、こどもを切替
  • 大人向けダッシュボードと AI生活サポートは、light / dark theme の両方で補助文字・グラフラベル・入力欄の可読性を確保

目標タブ内:

  • GenerationGoals: 画面上部で「個人 / こども / シニア」を切替
  • GoalsAndDebt: ローン残高・個人目標（「個人」タブ）の一覧と進捗管理
  • PurchaseAdvisor: 購入アドバイザー（同タブ内に表示）

4.3 設定

  • デザイン設定
  • アカウント設定

4.4 問い合わせ

  • 名前、メール、件名、本文を送信

6. AI 店案内仕様

入力:

  • 現在地またはエリア文字列
  • 店種
  • 自由入力キーワード
  • 半径

出力:

  • 店舗名
  • 住所
  • 距離
  • 緯度経度
  • 地図遷移リンク

検索仕様:

  • エリア入力時はサーバーで Google Geocoding API（または Nominatim フォールバック）でジオコーディング
  • 店舗検索は Google Places API（New）を優先し、失敗または 0 件の場合は Overpass API（OSM）へ自動フォールバック
  • Overpass 障害時は予備エンドポイント（overpass.kumi.systems）へ自動切り替え
  • 近い順に 6 件まで返却
  • 0 件時はエラーコードと検索座標を診断情報としてレスポンスに含む

エラー仕様:

  • エリア不明時は 404
  • ジオコーディング失敗時は 502
  • 不正入力時は 400

7. 購入アドバイザー仕様

入力:

  • カテゴリ（daily / appliance / beauty / game / oshi / car / home）
  • 購入価格（円）

計算出力:

  • 労働日数: 価格 ÷ (手取り月収 ÷ 30)
  • 貯蓄月数: 価格 ÷ 月次貯蓄額
  • ローン月額（car / home のみ、PMT 公式）
  • 購入可能ライン判定（アフォーダビリティ）

アフォーダビリティ閾値:

  • home: ローン月額 ≤ 手取り月収の 25%
  • car: ローン月額 ≤ 手取り月収の 20%
  • その他: 購入価格 ≤ 月次余剰（収入 − 支出）

目標連携:

  • 「目標に追加」ボタンで kakeibo-personal-goals（localStorage）へ保存
  • 保存形式: { id, name, emoji, targetAmount, currentAmount: 0, memo, status: "active", createdAt }

8. セキュリティ仕様

  • 認証済みユーザーのみ個人データを更新可能
  • same-origin 対象 API は cross-site 送信を拒否
  • rate limit 対象 API は短時間の多重送信を制限
  • 外部向け webhook は署名検証を必須化

セキュリティ評価: 93%（2026-04-26 実施、postcss override で npm audit 0 件）

9. 品質基準

リリース前に満たす基準:

  • eslint エラー 0
  • TypeScript エラー 0
  • next build 成功
  • 重大なハードコード秘密情報 0

10. 運用ルール

  • 実シークレットは Git に含めない
  • .env.local と Vercel 環境変数を分離する
  • DB migration は明示的に適用する

設計書

設計書

1. システム概要

本システムは、家計の記録、分析、予算管理、AI 支援を 1 つの Web アプリで提供する Next.js ベースの家計簿です。 フロントエンドと API を同一リポジトリの App Router で管理し、データは Supabase、認証は Auth0、配信は Vercel を利用します。

2. 全体構成

Browser
  ├─ App Router UI
  ├─ Dashboard / Input / Settings / Contact
  └─ Fetch to /api/*

Next.js App Router
  ├─ src/app/*                画面
  ├─ src/app/api/*            API Route Handlers
  ├─ src/lib/components/*     UI コンポーネント
  ├─ src/lib/auth/*           認証連携
  ├─ src/lib/server/*         セキュリティ / サーバーユーティリティ
  └─ src/lib/supabase/*       DB アクセス

External Services
  ├─ Supabase                 profiles / transactions / budgets
  ├─ Auth0                    Google / LINE / メール認証
  ├─ OpenAI / Gemini 等       AI 機能
  ├─ Google Maps API          周辺店舗検索 / Geocoding（優先）
  ├─ Nominatim (OSM)          エリアジオコーディング（フォールバック）
  ├─ Overpass API (OSM)       周辺店舗検索（フォールバック）
  ├─ Resend                   メール送信
  └─ Vercel                   Hosting / Build / Deploy

8. AI 店案内設計

対象:

  • src/lib/components/NearbyShopGuide.tsx
  • src/app/api/nearby-shops/route.ts

処理フロー:

  1. 画面で店種、エリア、自由入力キーワードを指定
  2. エリア入力時: /api/nearby-shops に送信 → Google Geocoding API（失敗時は Nominatim）でジオコーディング
  3. 現在地使用時: ブラウザの Geolocation API で座標を取得 → /api/nearby-shops に送信
  4. サーバーで Google Places API（New）を呼び出し（GOOGLE_MAPS_API_KEY 設定時）
  5. Google が失敗または 0 件の場合、サーバーから Overpass API（OSM）でフォールバック検索
  6. 距離順に最大 6 件へ整形し表示
  7. 0 件時はエラーコードと検索座標を診断情報としてレスポンスに付与し UI に表示

設計意図:

  • サーバーが認証・レート制限・ジオコーディング・店舗検索をすべて担当
  • Google Maps API が有効な場合は優先して使用し、失敗または 0 件時は OSM へ自動フォールバック
  • OSM（Nominatim / Overpass）は無料・APIキー不要で常に予備として機能
  • Overpass 障害時は予備エンドポイント（overpass.kumi.systems）へ自動切り替え
  • 0 件時は Google/OSM のエラーコードと検索座標を診断情報として返し、原因の特定を容易にする

9. 購入アドバイザー設計

対象:

  • src/lib/components/PurchaseAdvisor.tsx

機能:

  • 商品カテゴリ（日用品 / 家電・スマホ / 美容・ファッション / ゲーム / 推し活 / 車 / 住宅）を選択
  • 購入価格と AI 相場クエリを入力して相場を確認
  • 購入に必要な「労働日数」と「貯蓄月数」を数値で表示
  • 家・車はローン月額（PMT 公式）を計算し、収入に占める割合で購入可能ラインを判定
  • その他カテゴリは残余貯蓄で購入可能ラインを判定

ローン計算式（PMT 公式）:

月払い = P × r(1+r)^n / ((1+r)^n - 1)
住宅: 35 年 / 年利 1%
車:   5  年 / 年利 3%

アフォーダビリティ閾値:

  • 住宅ローン: 月額 ≤ 手取り月収の 25%
  • 車ローン: 月額 ≤ 手取り月収の 20%
  • その他: 価格 ≤ 月次余剰（収入 − 支出）

目標連携:

  • 目標画面は GenerationGoals の「個人 / こども / シニア」切替と、GoalsAndDebt の詳細タブで構成
  • 購入アドバイザーで「目標に追加」ボタンを押すと kakeibo-personal-goals（ローカルストレージ）へ目標を保存
  • GoalsAndDebt コンポーネントが同キーを参照するため、目標画面の「個人」タブで進捗管理が可能
  • dark theme 時は目標画面の補助ラベルと小さな注記を明るめに補正し、可読性を確保
  • 大人ダッシュボードでは Charts Calendar AIChat FoodLifestyleAssistant を含むカード群の文字色・グラフラベル・入力欄コントラストを補正

取扱説明書

取り扱い説明書

1. はじめに

このアプリは、収支の記録、予算管理、AI 相談、生活支援、購入アドバイスをまとめて行う家計簿です。

2. ログイン方法

利用できるログイン方法:

  • Google
  • LINE
  • メール

ログイン後はホームからダッシュボードへ進みます。

3. 基本的な使い方

3.1 収支を入力する

  1. 入力タブを開く
  2. 種別を選ぶ
  3. 金額、日付、カテゴリを入力する
  4. 必要に応じてメモや固定費設定を入れる
  5. 保存する

対応カテゴリ（支出）: 食費 / 住居 / 水道・光熱費 / 通信費 / 交通費 / 医療費 / 日用品 / 娯楽 / レジャー / 趣味 / 教育 / 自己投資 / 保険 / 税金 / 交際費 / サブスク / ペット / 美容・衣服 / ゲーム / 推し活 / 寄付・支援 / その他

3.2 ダッシュボードを見る

確認できる内容:

  • 当月の収支
  • 予算との差
  • カテゴリごとの使い方
  • 固定費見直し候補
  • 安全度と生活レベル
  • dark theme でもボード・カレンダー・AI サポートの補助文字やグラフラベルが読みやすいように調整されています

3.3 予算を設定する

  1. プリセット設定を開く
  2. 収入や配分を入力する
  3. 固定費、変動費、貯蓄、投資の割合を決める
  4. 保存してダッシュボードに反映する

4. AI 機能の使い方

4.1 AI 分析

  • 家計の状態を文章で確認したいときに使います

4.2 AI チャット

  • 家計について質問したいときに使います

4.3 AI 店案内

  1. 店種を選ぶ
  2. エリアを入力するか、現在地検索を使う
  3. 必要なら自由入力キーワードを入れる
  4. 検索結果の一覧と地図を確認する

使いどころ:

  • 食料品や日用品の買い物先を探したいとき
  • 生活圏の候補店を整理したいとき
  • 予算に合わせて買い物先を絞りたいとき

5. 購入アドバイザーの使い方

ダッシュボードの「目標」タブ内にある「購入アドバイザー」で大きな買い物の判断ができます。

5.1 基本の流れ

  1. カテゴリを選ぶ（日用品 / 家電・スマホ / 美容・ファッション / ゲーム / 推し活 / 車 / 住宅）
  2. 購入予定の金額を入力する
  3. 「判定する」を押す

5.2 表示される情報

  • 労働日数: その金額を稼ぐのに必要な日数（手取り月収÷30 で計算）
  • 貯蓄月数: 毎月の貯蓄をそのまま続けた場合に貯まるまでの月数
  • 購入可能ライン: 今の収支で無理なく買えるかどうかの判定

    • - 住宅・車はローン月額が収入の一定割合以内かどうかで判定 - その他は月次余剰（収入−支出）と比較して判定

5.3 相場を調べる

  • 「相場を調べる」欄に商品名や店名を入力して送信すると AI が市場価格の目安を教えてくれます

5.4 目標に追加する

  • 目標画面の上段で「個人」を選ぶと、個人向けの目標カードと進捗欄が表示されます
  • 「目標に追加」ボタンを押すと「目標」画面の「個人」タブにある個人目標リストへ追加されます
  • 追加後は進捗金額を更新して目標達成を管理できます

5.5 住宅・車のローン計算について

  • 住宅: 35 年返済 / 年利 1% で月額を計算し、手取りの 25% 以内を目安にしています
  • 車: 5 年返済 / 年利 3% で月額を計算し、手取りの 20% 以内を目安にしています

ログインできない

  • Auth0 / Google / LINE の設定値を確認する

現在地で店案内が使えない

  • ブラウザのアドレスバー横のアイコンから位置情報を許可する
  • 通信状況を確認する
  • HTTPS 環境でのみ位置情報が利用可能

エリア検索で店が見つからない

  • 駅名や市区町村名に変える
  • 自由入力キーワードを追加する
  • 時間をおいて再試行する（OpenStreetMap サーバーが混雑している場合あり）

周辺のお店情報を取得できない

  • しばらく待ってから再試行する
  • 別の店種や小さめのエリアで試す

購入アドバイザーで金額が表示されない

  • プリセット設定で手取り月収と月次貯蓄額を登録してください
  • 登録後は再度「判定する」を押してください

保存できない

  • セッション切れの可能性があります。再ログインしてください
